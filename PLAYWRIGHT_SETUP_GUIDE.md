# HostPulse Playwright Testing Quick Start Guide\n\n## Installation\n\n### Prerequisites\n- Node.js 18+ or Bun 1.0+\n- Git\n\n### Setup\n```bash\n# Install dependencies\nbun install\n\n# Install Playwright browsers\nbun run \"npx playwright install\"\n```\n\n## Running Tests\n\n### Local Testing\n```bash\n# Run all tests (all browsers)\nbun run test:e2e\n\n# Run specific browser\nbun run test:e2e -- --project=chromium\nbun run test:e2e -- --project=firefox\nbun run test:e2e -- --project=webkit\nbun run test:e2e -- --project=iphone\nbun run test:e2e -- --project=pixel\n\n# Run specific test file\nbun run test:e2e -- tests/e2e/analytics.spec.ts\n\n# Run tests in UI mode (interactive)\nbun run test:e2e:ui\n\n# Watch mode\nbun run test:e2e -- --watch\n\n# Debug single test\nbun run test:e2e -- --debug tests/e2e/auth.spec.ts\n\n# Re-run last failed tests\nbun run test:e2e -- --last-failed\n```\n\n### Viewing Results\n```bash\n# Open HTML report\nnpx playwright show-report\n\n# View trace for a specific test\nnpx playwright show-trace test-results/path-to-trace.zip\n```\n\n## Test Structure\n\n### Authentication Tests\n```bash\nbun run test:e2e -- tests/e2e/auth.spec.ts\n```\nCovers: Registration, Login, Logout\n\n### Property Management Tests\n```bash\nbun run test:e2e -- tests/e2e/properties.spec.ts\n```\nCovers: CRUD operations, Room management\n\n### Booking Management Tests\n```bash\nbun run test:e2e -- tests/e2e/bookings.spec.ts\n```\nCovers: Booking CRUD, Guest management, Filtering\n\n### Analytics & Dashboard Tests\n```bash\nbun run test:e2e -- tests/e2e/analytics.spec.ts\nbun run test:e2e -- tests/e2e/dashboard.spec.ts\n```\nCovers: Plan gating, Analytics features, Reports\n\n### Settings & Plan Management Tests\n```bash\nbun run test:e2e -- tests/e2e/settings.spec.ts\n```\nCovers: Profile updates, Plan management, Notifications\n\n### Cross-Browser Tests\n```bash\nbun run test:e2e -- tests/e2e/cross-browser.spec.ts\n```\nCovers: Chromium, Firefox, WebKit compatibility, Responsive design\n\n### Mobile Tests\n```bash\nbun run test:e2e -- tests/e2e/mobile.spec.ts\n```\nCovers: iPhone 15, Pixel 8, Touch interactions, Mobile forms\n\n### Performance Tests\n```bash\nbun run test:e2e -- tests/e2e/performance.spec.ts\n```\nCovers: Load times, Lazy loading, Network resilience\n\n## Writing New Tests\n\n### Basic Test Template\n```typescript\nimport { test, expect } from \"@playwright/test\";\nimport { installMocks } from \"./fixtures/mock-auth\";\n\ntest.describe(\"Feature Name\", () => {\n  test(\"should do something\", async ({ page }) => {\n    // Setup\n    await installMocks(page, { plan: \"professional\" });\n    await page.goto(\"/feature-path\");\n\n    // Action\n    await page.getByRole(\"button\", { name: /submit/i }).click();\n\n    // Assert\n    await expect(page.getByText(/success/i)).toBeVisible();\n  });\n});\n```\n\n### Using Mocks\n```typescript\n// Install mocks before navigation\nawait installMocks(page, { plan: \"professional\" });\n\n// Available plans: \"starter\", \"professional\", \"business\", \"enterprise\"\n// Custom analytics payload:\nawait installMocks(page, {\n  plan: \"business\",\n  analytics: {\n    occupancyRate: 0.85,\n    revenue: 50000,\n    bookings: 42,\n  },\n});\n```\n\n### Waiting Strategies\n```typescript\n// Wait for elements\nawait page.getByRole(\"button\", { name: /submit/i })\n  .waitFor({ state: \"visible\", timeout: 10000 });\n\n// Wait for URL change\nawait page.waitForURL(/dashboard/);\n\n// Wait for network idle\nawait page.waitForLoadState(\"networkidle\");\n\n// Wait for specific elements to load\nawait page.getByRole(\"heading\", { name: \"Analytics\" })\n  .waitFor({ state: \"visible\" });\n```\n\n## CI/CD Integration\n\nTests automatically run on:\n- Push to `main` branch\n- Pull requests\n- Manual workflow trigger via GitHub Actions\n\n### Viewing CI Results\n1. Go to Actions tab in GitHub\n2. Click on \"E2E Tests\" workflow run\n3. View logs or download artifacts:\n   - `playwright-report-{browser}` - HTML test report\n   - `test-results-{browser}` - JUnit XML results\n\n## Debugging Failed Tests\n\n### View Failure Details\n```bash\n# Run single failing test in debug mode\nbun run test:e2e -- --debug tests/e2e/analytics.spec.ts -g \"professional\"\n\n# View HTML report for details\nnpx playwright show-report\n```\n\n### Check Artifacts\n- Screenshots: `test-results/**/test-failed*.png`\n- Videos: `test-results/**/video.webm`\n- Traces: `test-results/**/trace.zip`\n\n### Common Issues\n\n**Tests timeout on CI but pass locally**\n- Increase timeout in `playwright.config.ts`\n- Check for slow network requests\n- Review performance test results\n\n**Flaky tests (pass/fail randomly)**\n- Review wait strategies - ensure robust waits\n- Check for race conditions\n- Verify mock setup is complete before navigation\n\n**Mobile tests fail**\n- Check viewport size matches device configuration\n- Verify touch event handling\n- Test on actual devices using BrowserStack/Sauce Labs\n\n## Performance Expectations\n\n| Scenario | Expected Time |\n|----------|---------------|\n| Single test | 5-30s |\n| Single browser suite | 2-3 min |\n| All desktop browsers | 5-8 min |\n| All browsers (desktop + mobile) | 10-15 min |\n| With CI retries | 15-20 min |\n\n## Best Practices\n\n1. **Use Page Object Model for complex apps**\n   ```typescript\n   class DashboardPage {\n     constructor(private page: Page) {}\n     async goto() { await this.page.goto('/dashboard'); }\n     async getOccupancyValue() { return this.page.getByText(/occupancy/i); }\n   }\n   ```\n\n2. **Always wait before asserting**\n   ```typescript\n   // ❌ Wrong\n   const btn = page.getByRole(\"button\");\n   await expect(btn).toBeEnabled();\n\n   // ✅ Correct\n   const btn = page.getByRole(\"button\");\n   await btn.waitFor({ state: \"visible\" });\n   await expect(btn).toBeEnabled();\n   ```\n\n3. **Use semantic locators**\n   ```typescript\n   // ✅ Good - accessible labels\n   page.getByLabel(/email/i)\n   page.getByRole(\"button\", { name: /submit/i })\n   \n   // ❌ Avoid - brittle selectors\n   page.locator(\".email-input\")\n   page.locator(\"button.submit\")\n   ```\n\n4. **Set up before each test**\n   ```typescript\n   test.beforeEach(async ({ page }) => {\n     await installMocks(page, { plan: \"professional\" });\n     await page.goto(\"/dashboard\");\n   });\n   ```\n\n5. **Use meaningful test names**\n   ```typescript\n   // ✅ Clear\n   test(\"professional plan unlocks 7d/30d but locks 90d and YTD\", ...)\n   \n   // ❌ Vague\n   test(\"should work\", ...)\n   ```\n\n## Resources\n\n- [Playwright Documentation](https://playwright.dev)\n- [Testing Best Practices](https://playwright.dev/docs/best-practices)\n- [Debugging Guide](https://playwright.dev/docs/debug)\n- [API Reference](https://playwright.dev/docs/api/class-page)\n\n## Support\n\nFor issues or questions:\n1.
