# HostPulse AI Capability Audit

_Generated by codebase survey. All paths relative to repo root._

Scope: maps every existing AI, search, recommendation, pricing, analytics, discovery, concierge, fraud, vision, automation, executive-HQ, and market-intelligence capability to the 15 phases of the "HostPulse AI Enterprise Integration" brief. Nothing has been changed in code — this is the "audit only" deliverable so the next batch of work can be scoped.

Legend for phase mapping (later in this doc):
- ✅ Already Complete
- 🟡 Partially Complete
- ❌ Missing

## Cross-Cutting Findings

### pgvector / PostGIS / Embeddings / Redis / Python
- **pgvector**: ❌ Not enabled. No `vector` column, no `<->` operator, no embedding storage.
- **PostGIS**: ❌ Not installed. `latitude` / `longitude` are plain `numeric`.
- **Embeddings**: ❌ None stored anywhere.
- **Redis**: ❌ None (Supabase table used for rate-limit windows).
- **Python / FastAPI**: ❌ None (all TS/TanStack on Cloudflare Workers).
- **Supabase Edge Functions**: only `mpesa-callback` (payments, not AI).

### ♻️ Duplicate modules / merge candidates

| Issue | Files | Recommendation |
|---|---|---|
| AI gateway duplicated | `discovery.server.ts` + `onboarding-ai.functions.ts` both re-implement the gateway fetch instead of importing `ai.server.ts`. | Refactor both to use `aiChat` / `aiJSON`. |
| Fraud split | `advanced-analytics.functions.ts::fraudSignals()` (heuristics) vs `admin-ops.functions.ts::adminFraudOverview` (rate-limit view). | Consolidate under a single `fraud.functions.ts`. |
| Occupancy calc duplicated | `command-center.functions.ts`, `revenue-intelligence.functions.ts::forecastOccupancy`, `analytics.functions.ts`. | Extract `occupancyUtils.ts`. |
| Search-event logging duplicated | Local `logSearch()` in `concierge.functions.ts` vs `logSearchEvent` server fn in `knowledge.functions.ts`. | Drop the local copy; call the server fn. |

### Recommended next-batch scope (post-audit)

If we ship next-phase work in order of leverage per unit of effort:

1. **Consolidate the 4 ♻️ duplication items** — small change, unblocks everything else.
2. **Phase 3 (semantic search)** — enable `pgvector`, add embeddings column on `marketplace_properties` + `discovered_properties`, backfill via `google/gemini-embedding-2`, replace `ilike` retrieval in concierge + marketplace search.
3. **Phase 4 (recommendations)** — event capture (clicks, saves, bookings) → simple co-visit + content-based ranker over embeddings from step 2.
4. **Phase 8 (vision) via LLM vision** — replace byte-hash with `google/gemini-2.5-flash` image tagging for amenities/rooms + confidence; keep byte-hash as dedupe fast path.
5. **Phase 10 (review NLP)** — batch `aiJSON` sentiment + aspect scoring, stored as `knowledge_property_facts` scope=`reviews`.
6. **Phase 14 (market intelligence)** — nightly rollup tables (`county_market_stats`, `heatmap_cells`) fed from marketplace + discovered + reservations.

Each of the above is a single build turn of comparable size to the existing modules.
